## What this product is

A hard fork of Fabro (`fabro-sh/fabro`, MIT) rebranded and owned as **pixl-factory**
(`github.com/nuva-be/pixl-factory`). Goal: own the workflow engine, embed the pixl
stack (crew + kb), and make it a **daily-driver tool** run on the Claude subscription —
eventually self-hosted on a VPS. Engines stay headless; pixl-factory is the one UI.

## The three differentiators (all proven, on the subscription)

1. **Execution** — pixl-factory runs DOT (`.fabro`) workflows; we added a native `kb` backend.
2. **Crew** — `backend="acp"` loads the full pixl-crew plugin (14 agents, ~117 skills) into Claude Code.
3. **Memory** — pixl-kb reachable 3 ways: MCP tool inside ACP nodes, the auto run-report hook, and the native `kb` node. Verified: the kb node read `doc-925b8649`, which the memory hook wrote earlier in the same session.
